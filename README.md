## Описание проекта

Данный проект реализует **параллельный алгоритм умножения квадратных матриц** с использованием технологии **OpenMP** на языке **C++**.

# Структура проекта

```
.
├── matrix_mul_omp.cpp        # Основная программа (OpenMP)
├── matrix.py                 # Скрипт проверки
├── experiment_results.txt    # Результаты экспериментов
├── verification_report.txt   # Отчёт проверки
├── matrix_a_<N>.txt          # Матрица A
├── matrix_b_<N>.txt          # Матрица B
├── matrix_c_<N>.txt          # Результат умножения
└── README.md                 # Документация
```

---

# Компиляция и запуск

##  Компиляция программы

Используется компилятор **MSVC** с поддержкой OpenMP:

```bash
cl /openmp matrix_mul_omp.cpp
```
---

##  Запуск программы

```bash
matrix_mul_omp.exe
```

После запуска программа:

1. генерирует случайные матрицы
2. выполняет параллельное умножение
3. измеряет время выполнения
4. сохраняет результаты в файлы

---

## 3️ Проверка корректности результатов

Для проверки используется Python-скрипт:

```bash
python matrix.py
```

Скрипт:

1. считывает матрицы `A`, `B`, `C`
2. вычисляет эталонное произведение

```python
np.dot(A, B)
```

3. сравнивает результаты
4. формирует файл **verification_report.txt**

# Анализ производительности

Теоретическая сложность алгоритма:

```
O(N³)
```

Наблюдения:

* При увеличении размера матрицы **в 2 раза** время увеличивается примерно **в 8 раз**.
* Это соответствует теоретической сложности `O(N³)`.
* На больших размерах матриц появляются отклонения из-за:

  * ограничений **кеш-памяти процессора**
  * затрат на **доступ к памяти**
  * особенностей **параллельного выполнения**
 * График зависимости Времени от объема матрицы

Использование **OpenMP** позволяет эффективно задействовать **все ядра процессора**, что значительно ускоряет вычисления.
