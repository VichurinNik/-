# Параллельное умножение матриц (MPI)

##  Описание
Данный проект реализует алгоритм **параллельного умножения квадратных матриц** на языке **C++** с использованием двух технологий:
* **MPI (Message Passing Interface)** — модель распределённой памяти

---

##  Возможности
* Параллельное умножение матриц
* Проведение серии экспериментов
* Несколько прогонов для повышения точности
* Расчёт:
  * времени выполнения
  * ускорения (Speedup)
  * эффективности
---

## Параметры эксперимента

* Размеры матриц:

```
200, 400, 800, 1200, 1600, 2000
```
* Количество прогонов: **3**
* MPI процессы: **1, 2, 4**

# Сравнение MPI и OpenMP

| Критерий           | MPI               | OpenMP                |
| ------------------ | ----------------- | --------------------- |
| Модель памяти      | Распределённая    | Общая                 |
| Сложность          | Высокая           | Низкая                |
| Обмен данными      | Через сообщения   | Не требуется          |
| Масштабируемость   | Высокая (кластер) | Ограничена (1 машина) |
| Производительность | Средняя           | Высокая               |

# Анализ результатов

##  MPI

* Ускорение достигает **~2.7x на 4 процессах**
* Есть накладные расходы:

  * передача данных между процессами
  * синхронизация
* Эффективен для **распределённых систем и больших задач**

---

##  OpenMP

* Ускорение достигает **до 10x (8 потоков)**
* Практически отсутствуют накладные расходы
* Максимально эффективно использует ресурсы одного ПК

---

##  Ключевое различие

* **MPI** тратит время на обмен данными
* **OpenMP** работает с общей памятью → быстрее

---

#  Выводы

* **OpenMP значительно быстрее MPI** на одном компьютере
*  MPI лучше подходит для **кластеров и распределённых систем**
*  При увеличении размера матриц:

  * эффективность MPI растёт
  * OpenMP показывает максимальную производительность

Итог:

* **OpenMP — лучший выбор для одного ПК**
* **MPI — лучший выбор для многомашинных систем**
