# Параллельное умножение матриц с OpenMP

Данный проект реализует алгоритм умножения квадратных матриц с использованием технологии параллельных вычислений OpenMP на языке C++. Программа проводит серию вычислительных экспериментов для матриц различного размера, замеряет время выполнения и автоматически сохраняет результаты.

## Возможности

- Генерация случайных квадратных матриц заданного размера
- Параллельное умножение матриц с использованием OpenMP
- Замер времени выполнения с помощью `omp_get_wtime()`
- Сохранение исходных и результирующих матриц в текстовые файлы
- Проведение экспериментов для матриц размером: 200, 400, 800, 1200, 1600, 2000
- Автоматическая верификация результатов с помощью Python и NumPy
- Формирование отчёта с результатами в удобном формате

## Структура проекта
├── matrix_mul_omp.cpp # Основная программа на C++ (OpenMP)
├── matrix.py # Скрипт для верификации результатов
├── experiment_results.txt # Итоговая таблица с результатами замеров
├── verification_report.txt # Отчёт о проверке корректности
├── matrix_a_<N>.txt # Сгенерированная матрица A (для каждого N)
├── matrix_b_<N>.txt # Сгенерированная матрица B (для каждого N)
├── matrix_c_<N>.txt # Результирующая матрица C (для каждого N)
└── README.md # Описание проекта
Компиляция и запуск
1. Компиляция C++ программы
cl /openmp matrix_mul_omp.cpp
2. Запуск программы
matrix_mul_omp.exe     
3. Верификация результатов
python matrix.py
Результаты экспериментов
После выполнения программы в файле experiment_results.txt будет сохранена таблица:
Размер матрицы	Время выполнения (сек)
200              	0,020636
400	              0,161870
800              	1,297605
1200            	4,543726
1600            	27,659698
2000            	68,863224
Верификация
Скрипт matrix.py выполняет автоматическую проверку:

Читает матрицы A, B и C из файлов

Вычисляет эталонное произведение с помощью np.dot(A, B)

Сравнивает результаты с матрицей C из программы

Формирует подробный отчёт
Анализ производительности
Теоретическая сложность алгоритма: O(N³)

Экспериментальные наблюдения:

При увеличении размера матрицы с 400 до 800 (в 2 раза) время выполнения увеличилось примерно в 8 раз, что соответствует теоретической сложности

На больших размерах (1600, 2000) наблюдаются отклонения, связанные с работой кеш-памяти процессора
Параллельные вычисления на всех ядрах процессора позволяют эффективно обрабатывать матрицы большого размера
Примечания
Все сгенерированные матрицы сохраняются в текущей директории
При повторном запуске программы старые файлы перезаписываются
Для корректной работы верификации все файлы matrix_*_<N>.txt должны находиться в одной папке со скриптом
