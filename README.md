#

Данный проект реализует параллельное умножение квадратных матриц с использованием технологии **CUDA**.  
Программа автоматически генерирует матрицы, выполняет вычисления на GPU и проводит серию бенчмарков для различных размеров матриц и размеров блоков.
## Цель работы
Изучить эффективность параллельных вычислений на GPU и влияние размера блока (`block size`) на производительность умножения матриц.
---
## Возможности программы

- Генерация случайных квадратных матриц
- Чтение и запись матриц в файлы
- Параллельное умножение на GPU
- Автоматический бенчмарк
- Измерение времени выполнения
- Подсчет производительности (GFLOPS)
- Сохранение результатов в CSV
---
## Алгоритм

Каждый поток CUDA вычисляет один элемент результирующей матрицы:

C[i][j] = sum(A[i][k] * B[k][j])

Используется двумерная сетка потоков (grid) и блоков (block).

### Производительность (GFLOPS)

| Размер | 8x8 | 16x16 | 32x32 |
|--------|--------|--------|--------|
| 200    | 296.91 | 279.05 | 186.89 |
| 400    | 373.14 | 367.51 | 326.38 |
| 800    | 351.95 | 351.39 | 347.90 |

| 2000   | 367.67 | 362.38 | 367.12 |

---

## Анализ результатов

- На малых матрицах (200x200) лучший результат показывает блок 8x8
- Блок 32x32 хуже работает на маленьких данных
- На больших размерах (800+) разница почти исчезает
- Производительность стабилизируется около 350–370 GFLOPS
- Оптимальные размеры блока: 8x8 и 16x16
